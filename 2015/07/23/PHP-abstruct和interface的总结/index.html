PHP abstruct和interface的总结

Posted on 2015-07-23 | Edited on 2015-10-31 | In php | Comments: | Views:

经历了几次面试，经常被问到php的abstruct和interface，因为没有系统的总结过，所以答的都不算很好。正好在看《PHP高级程序设计模式，框架与测试》这本书也提到了这两个特性，所以总结下：

抽象类

abstruct class 机制中总是要定义一个公共的base class，特写的细节让继承者来实现。也就是说，当需要一个基础对象，而创建的某些方法没有被完整的定义的时候，就要用到抽象类。

  • 一个类只要包含一个抽象方法就必须声明为抽象类。
  • 声明为抽象的方法，在实现的时候必须包含相同或更低的访问级别。
  • 不能用new关键字来创建抽象类的实例。
  • 被声明为抽象的方法不能包含函数体。
  • 继承一个抽象类就必须实现它的所有抽象方法。

接口

接口是一种类似于类的结构，可用于声明实现类所必须声明的方法。

接口只能包含方法原型(prototype)，而不能包含任何完整定义了的方法

方法的原型是指方法的定义中剔除了方法体之后的签名。

给定的实现类上可以使用多个接口。类必须实现接口的所有方法。

可用instanceof操作符来确定类的继承者是否实现接口。

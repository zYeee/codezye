docker搭建并部署LNMP环境(一)

Posted on 2016-06-06 | In linux | | Visitors:

　　作为开发一直都想要一个自动化的集成环境解决方案，这样可以省去很多部署环境的时间，尽可能消除不同操作系统之间的差异。为了达到这个目的以前一直用的都是vagrant这套工具，但是vagrant从本质上说还是操作系统级别的虚拟机，只适合跑在本地。而且在繁琐地配置了几台vps后，更迫切地需要一个更轻量级的方案来优化一些开发工作。
　　docker作为一个能够提供容器级别的虚拟化的工具，可以很好的达到我的目的。docker作为一个前几年的新兴技术已经有越来越火的趋势，也有越来越多的公司把docker用于生产环境。于是就有了一个动手搭建一套基于docker的LNMP环境，下面做一些简单的记录和介绍：
　　docker官方所推崇的理念是单一进程容器，是与传统的把多个服务放在一个宿主机内的概念相对的一个思想。所以这一套最简单的LNMP架构，也可以拆分成三个不同的容器，并进行连接。官方也提供了相应的镜象。

运行一个容器的时候还有一个注意点就是数据卷挂载，因为在本地开发的时候项目代码应该是放在本地目录，这样才能进行实时调试。docker有两种方式可以挂载本地数据卷。

  • 在Dockerfile里添加VOLUME[]命令挂载本地数据卷
  • 执行docker run指令的时候通过-v参数添加本地数据卷挂载

这两种方式是有区别的，我研究了一下：
在Dockerfile里加入VOLUME[‘/tmp’]后运行容器，通过inspect发现，mounting信息是这样的：

docker run 的时候加入-v /tmp:/tmp参数，inspect it:

从这里可以看出：Dockerfile里的VOLUME是把宿主机的/mnt/sda1/var/lib/docker/volumes/…文件夹挂载到container中去，并不能在宿主机上修改，但是能作为一个本地卷可以让多个声明挂载了该卷的cantainer共享。
而docker run的-v参数就可以使容器挂载指定的本地目录。这个目录的文件可以在宿主机上被修改，也可以被所有的container所共享。
